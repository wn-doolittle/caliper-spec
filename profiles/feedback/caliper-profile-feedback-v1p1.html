Abstract

This document extends Caliper 1.1 by defining a Informal Feedback Profile that describes informal feedback related learning activities.

Overview

Design Goals and Rationale

The Caliper Informal Feedback Profile models a Person providing informal feedback on a Entity, typically a piece of student work. The profile provides a InformalFeedbackEvent along with 'Commented' and 'Ranked' actions for describing feedback activities. The Rating entity is used to capture quantitative feedback while the Comment entity is used to capture qualitative feedback.

User Stories

The Informal Feedback Profile supports the following learner-centric user stories:

As an instructor, I would like an overview of the overall quality of peer feedback my students gave to each other so I can make informed enhancements to my instructions.
As a student, I would like to see the common shortcomings and merits in my work over a term/year based on peer and instructor comments in order to gain insight on what areas I need to improve upon.

Entities

The InformalFeedbackEvent MUST reference a Rating or Comment object that captures the content of the feedback.

JSON-LD

Individual Caliper events and entity describes are serialized as JSON-LD. IMS Global provides a remote Informal Feedback Profile JSON-LD context for mapping InformalFeedbackEvent terms to IRIs. Caliper message producers MUST reference this context in every Caliper 1.1 InformalFeedbackEvent, Rating, Comment, Scale, and/or LikertScale describe created and serialized for transmission to a target endpoint.

See the Caliper 1.1 specification [[!CALIPER-11]] for a discussion of JSON-LD and JSON-LD context handling.
