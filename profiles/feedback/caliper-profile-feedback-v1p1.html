Abstract

This document extends Caliper 1.1 by defining a Feedback Profile that describes informal feedback related learning activities.

Overview

Design Goals and Rationale

The Caliper Feedback Profile models a Person providing informal feedback on a Entity, typically a piece of student work. The profile provides a FeedbackEvent along with Commented and Ranked actions for describing feedback activities. The Rating entity is used to capture quantitative feedback while the Comment entity is used to capture qualitative feedback.

User Stories

The Feedback Profile supports the following learner-centric user stories:

As an instructor, I would like an overview of the overall quality of peer feedback my students gave to each other so I can make informed enhancements to my instructions.
As a student, I would like to see the common shortcomings and merits in my work over a term/year based on peer and instructor comments in order to gain insight on what areas I need to improve upon.

Events

The Feedback Profile is provisioned with a single a FeedbackEvent for describing informal feedback provided by an actor, typically a learner.

FeedbackEvent

A Caliper FeedbackEvent models a Person providing informal feedback on an Entity, typically a DigitalResource or another Person.

Term
FeedbackEvent
IRI
https://purl.imsglobal.org/caliper/FeedbackEvent
Supertype
Event
Properties
The FeedbackEvent inherits all properties defined by its supertype Event, of which id, type, actor, action, object, and eventTime are required. Additional profile-specific type and value restrictions are described below:
Property Type Description Disposition
type Term The string value MUST be set to FeedbackEvent. Required
actor Person | IRI The Person who initiated the action. The actor value MUST be expressed either as an object or as a string corresponding to the actor's IRI. Required
action Term The action or predicate that binds the actor or subject to the object. The value range is limited to the Commented and Ranked actions only. Required
object Entity | IRI The Entity that is the target of the feedback. The object value MUST be expressed either as an object or as a string corresponding to the resource's IRI. Required
target Frame | IRI If the object of the feedback is a particular segment of a DigitalResource use a Frame to mark its location. The target value MUST be expressed either as an object or as a string corresponding to the target entity‚Äôs IRI. Optional
generated Rating | Comment | IRI The Rating or Comment entity that describes the feedback provided. If the action is Ranked then the generated value MUST be expressed as a Rating. If the action is Commented then the generated value MUST be expressed as a Comment. The generated value MUST be expressed either as an object or as a string corresponding to the entity's IRI. Optional

Entities

The FeedbackEvent SHOULD reference a generated Comment or Rating that captures the content of the feedback.

A Caliper LikertScale models a Likert scale employed for scaling responses.

Properties

MultiselectScale

A Caliper MultiselectScale models a scale used to capture one or more rating options.

MultiselectScale
IRI
https://purl.imsglobal.org/caliper/MultiselectScale

Scale

MultiselectScale inherits all properties defined by its supertype Scale, of whichid, and type are required. MultiselectScale is also provisioned with six additional properties scalePoints, itemLabels, itemValues, orderedSelection, minSelections, and maxSelections. Profile-specific type restrictions are described below:
Property Type Description Disposition

itemValues Array [{key:value}] | Array [value] The ordered list of values for each point on the scale. Optional

minSelections integer Indicates the minimum number of selections that can be chosen. Optional

Expressing Caliper as JSON-LD

Individual Caliper events and entity describes are serialized as JSON-LD. IMS Global provides a remote Feedback Profile JSON-LD context for mapping FeedbackEvent terms to IRIs. Caliper message producers MUST reference this context in every Caliper 1.1 FeedbackEvent, Rating, Comment, Scale, LikertScale, MultiselectScale, and/or NumericScale describe created and serialized for transmission to a target endpoint.

See the Caliper 1.1 specification [[!CALIPER-11]] for a discussion of JSON-LD and JSON-LD context handling.
