Abstract

This document extends Caliper 1.1 by defining a Resource Management Profile that describes actions related to managing entities.

Overview

Design Goals and Rationale

The Caliper Resource Management Profile models a Person managing a Entity. The profile provides a ResourceManagementEvent with several actions describing how the resource is being managed: 'Archived', 'Copied', 'Created', 'Deleted', 'Described', 'Modified', 'Published', 'Restored', 'Retrieved', and 'Unpublished'. The 'Archived', 'Copied', 'Published', 'Restored', and 'Unpublished' actions are new actions

Events

The Resource Management Profile is provisioned with a single a ResourceManagementEvent for describing resource management related activities.

ResourceManagementEvent

A Caliper ResourceManagementEvent models a Person managing an Entity.

All actions require the 'id', 'type', 'actor', 'action', and 'object' properties. In addition the 'Copied' action requires the 'generated' property to store the newly generated Entity generated by the action.

id UUID The emitting application MUST provision the ResourceManagementEvent with a UUID. A version 4 UUID SHOULD be generated. The UUID MUST be expressed as a URN using the form 'urn:uuid:<UUID>' per [[rfc4122]]. Required

Actions

The Caliper ResourceManagementEvent supports the 'Archived', 'Copied', 'Created', 'Deleted', 'Described', 'Modified', 'Published', 'Restored', 'Retrieved', and 'Unpublished' actions.

Entities

The ResourceManagementEvent does not add any new entities.

JSON-LD

Individual Caliper events and entity describes are serialized as JSON-LD. IMS Global provides a remote Resource Management Profile JSON-LD context for mapping ResourceManagementEvent terms to IRIs. Caliper message producers MUST reference this context in every Caliper 1.1 ResourceManagementEvent describe created and serialized for transmission to a target endpoint.

See the Caliper 1.1 specification [[!CALIPER-11]] for a discussion of JSON-LD and JSON-LD context handling.
